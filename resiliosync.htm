BDIX P2P File Sharing Methods

Resilio Sync

  • Encrypted P2P via BitTorrent protocol.
  • Install on both PCs, add folder → Share → Generate link → Paste on peer.
  • Open a fixed port (e.g. 55555 TCP/UDP) via UPnP or manual forwarding.
  • BDIX tip: Manually add peer’s BDIX IP (e.g. tcp://10.x.x.x:55555).
  • Speed: Up to full ISP bandwidth (e.g. 90–95 Mbps on a 100 Mbps plan).
  • Pros: Simple GUI, cross‑platform, encrypted, direct P2P.
  • Cons: If NAT fails, falls back to Resilio relays (non‑BDIX).

Syncthing

  • Open‑source folder sync with web GUI (http://localhost:8384).
  • Add each other’s Device IDs, then share folders in GUI.
  • Forward TCP/UDP port 22000; optionally specify BDIX IP in “Addresses”.
  • Speed: Full BDIX link (e.g. multi‑MB/s on 100 Mbps).
  • Pros: Private, no cloud, conflict handling, encrypted.
  • Cons: Requires config; if misconfigured, uses public relays.

BitTorrent

  • Create a private .torrent (or magnet) with no external tracker.
  • Seed on A, download on B; exchange the .torrent file or magnet link.
  • Forward your client’s listening port (e.g. 6881 TCP/UDP).
  • BDIX tip: Disable DHT/public trackers, or run a local opentracker.
  • Speed: Saturates your line (e.g. 90–95 Mbps on 100 Mbps plan).
  • Pros: Robust, resumable, widespread clients.
  • Cons: IP/port exposure; NAT config needed; may leak off‑BDIX if DHT on.

Python HTTP Server

  • Run python3 -m http.server 8000 in the folder.
  • On peer, visit http://<A’s_BDIX_IP>:8000 to download files.
  • Forward TCP port 8000 on your router; open firewall.
  • Pros: No install on many systems; super‑quick setup.
  • Cons: No resume; no encryption (HTTP only).

FTP Server

  • Install FileZilla Server (Win) or vsftpd (Linux).
  • Configure user + folder; forward port 21 + passive range (e.g. 50000–50010).
  • Connect via any FTP client (FileZilla, browser).
  • Pros: Resume support; parallel transfers.
  • Cons: No encryption by default (use FTPS/TLS if needed).

iperf3 Speed Test

  • On A: iperf3 -s
  • On B: iperf3 -c <A’s_BDIX_IP> -P 4 (4 parallel streams)
  • Result shows raw TCP throughput over BDIX (e.g. 800 Mbps on gigabit links).
  • Tip: Close other network apps; repeat at different times to gauge load.
